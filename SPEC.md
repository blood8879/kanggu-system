### 시스템 설명
- [ ] 클라이언트 사이드 전용 웹 애플리케이션으로 근로계약서 자동 생성

- [ ] IT 리터러시가 높지 않을 수 있음
- [ ] 빠르고 간단한 프로세스 선호

## 문제 정의 및 해결책

### 현재 문제점
- [ ] (주)강구토건에서 신규 일용직 근로자마다 계약서 작성 필요
- [ ] 기존 근로자도 매달 1일마다 계약서 신규 작성
- [ ] 수기 작성으로 시간 소요 및 오류 발생 가능성

### 핵심 문제
- [ ] 반복 근로자의 계약서 작성 시간 낭비
- [ ] 복잡한 엑셀 양식 수기 입력 비효율성
- [ ] 정보 누락 또는 오입력 가능성

### 제안 해결책
- [ ] 웹 기반 직관적인 입력 폼
- [ ] 근로자 정보 자동 저장 및 불러오기
- [ ] 자동 계약서 생성 (엑셀)
- [ ] 즉시 다운로드 및 출력 가능

### 핵심 가치 제안
- [ ] 시간 절감: 작성 시간 80% 단축 (20분 → 4분)
- [ ] 오류 감소: 자동화로 입력 오류 최소화
- [ ] 확장성: 다양한 서류 양식 추가 가능
- [ ] 사용 편의성: 직관적인 UI
- [ ] 비용 효율성: 서버 비용 $0
- [ ] 개인정보 보호: 사용자 기기에만 저장

## 입력 항목 명세

### 엑셀 템플릿 구조
- [ ] **Before Template**: 단일 시트 (8시간(8hx6)_1월)
- [ ] **After Template**: 13개 시트 (12개월 + 요약)
- [ ] 크기: A1:S51 (19열 × 51행)

### 회사 정보 (사용자 입력)
- [ ] 회사명 (기본값: ㈜강구토건, 수정 가능)
- [ ] 대표자 (기본값: 이진호, 수정 가능)
- [ ] 본사주소 (기본값: 서울시 마포구 희우정로16, 8층, 수정 가능)
- [ ] 현장주소 (선택, 미입력 시 빈 값)
- [ ] 현장대리인 (선택, 미입력 시 빈 값)

### 근로자 정보 (배열 입력)
- [ ] **다중 근로자 입력 가능** (배열 구조)
  - [ ] 한 명 입력: 파일 1개 생성 및 다운로드
  - [ ] 여러 명 입력: 각 근로자별 파일 생성 및 다중 다운로드
- [ ] 각 근로자별 정보 (모두 선택 항목)
  - [ ] 성명 (선택, 미입력 시 빈 값)
  - [ ] 주민등록번호 (선택, 미입력 시 빈 값)
  - [ ] 주소 (선택, 미입력 시 빈 값)
  - [ ] 핸드폰번호 (선택, 미입력 시 빈 값)

### 근로 조건 (사용자 입력)
- [ ] 근로장소 (선택, 미입력 시 빈 값)
- [ ] 직종 (선택, 미입력 시 빈 값)
- [ ] 계약 시작일 (필수)
  - [ ] 종료일 자동 계산 (해당 월 말일)
  - [ ] 예: 2025-05-15 → 2025-05-31
- [ ] 일당 (기본값: 160,000원, 수정 가능)

## 기능 요구사항

### FR1: 근로자 정보 관리
- [ ] FR1.1: 신규 근로자 정보 입력
- [ ] FR1.2: 기존 근로자 정보 검색 (이름 또는 주민등록번호)
- [ ] FR1.3: 근로자 정보 수정
- [ ] FR1.4: 근로자 정보 저장 (IndexedDB)
- [ ] FR1.5: 저장된 근로자 정보 목록 조회 및 선택

### FR2: 계약서 생성
- [ ] FR2.1: 계약 시작일 기반 계약서 생성
  - [ ] 시작일 선택 시 해당 월 시트 자동 선택
  - [ ] 종료일 자동 계산 (해당 월 말일)
- [ ] FR2.2: 입력 정보를 엑셀 템플릿 셀에 자동 입력
- [ ] FR2.3: 수식 포함 셀 자동 계산 (임금 관련)
- [ ] FR2.4: 다중 근로자 처리
  - [ ] 근로자 1명: 파일 1개 생성 및 단일 다운로드
  - [ ] 근로자 여러 명: 각 근로자별 파일 생성
  - [ ] 다중 파일 자동 다운로드 (순차 또는 ZIP 압축)
- [ ] FR2.5: 브라우저에서 엑셀 파일 생성 및 다운로드

### FR3: 사용자 인터페이스
- [ ] FR3.1: 직관적인 입력 폼 제공
- [ ] FR3.2: 입력 항목 검증 (필수 항목, 형식)
- [ ] FR3.3: 이전 근로자 자동완성 기능
- [ ] FR3.4: 반응형 디자인 (모바일 접근성)

### FR4: 확장성
- [ ] FR4.1: 다양한 서류 양식 추가 가능한 구조
- [ ] FR4.2: 템플릿 관리 기능 (향후)
